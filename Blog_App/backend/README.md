BlogApp — Backend
Bun + Elysia + Drizzle ORM + PostgreSQL REST API.
Folder structure
```
src/
  db/
    schema.ts        # all tables + relations (1:1, 1:N, N:N)
    index.ts          # drizzle db instance
    migrate.ts         # runs generated SQL migrations
    seed.ts            # inserts demo data
    migrations/         # auto-generated by drizzle-kit
  lib/
    auth.ts             # bcrypt password hashing helpers
  middleware/
    auth.middleware.ts  # JWT decoding + isAuthenticated / isAdmin guards
  modules/
    auth/                # register, login, logout, /me
    users/                # profiles, admin user list
    categories/            # post categories
    tags/                   # post tags
    posts/                   # the core CRUD resource
    comments/                 # nested under /posts/:postId/comments
  index.ts                     # app entrypoint - wires everything together
```
Every module follows the same 3-file pattern:
`*.model.ts`   → request validation (Elysia's typebox `t.Object`)
`*.service.ts` → database access (Drizzle queries) — pure functions, no HTTP
`*.controller.ts` → Elysia routes — calls the service, shapes the HTTP response
1. Install prerequisites
Bun `curl -fsSL https://bun.sh/install | bash`
PostgreSQL running locally (or use Docker: `docker run --name blogapp-db -e POSTGRES_PASSWORD=postgres -p 5432:5432 -d postgres:16`)
2. Install dependencies
```bash
cd backend
bun install
```
3. Configure environment
```bash
cp .env.example .env
# edit .env if your DATABASE_URL / JWT_SECRET differ
```
4. Generate & run migrations
```bash
bun run db:generate   # reads schema.ts, writes SQL into src/db/migrations
bun run db:migrate    # applies the SQL to your database
```
5. (Optional) seed demo data
```bash
bun run db:seed
```
Creates `admin@example.com` / `Admin@123` (role: admin) and
`john@example.com` / `John@123` (role: user), plus a sample post with tags & a comment.
6. Run the dev server
```bash
bun run dev
```
API base: `http://localhost:4000`
Interactive Swagger docs (built into Elysia): `http://localhost:4000/docs`
API summary
Method	    Route	        Auth	Description
POST	    /auth/register	Public	Create account
POST	    /auth/login	    Public	Login, returns JWT
POST	    /auth/logout	Public	Clears auth cookie
GET	        /auth/me	   Logged in	Current user
GET	        /users	        Admin	List all users
GET	        /users/:id	    Public	Public profile (with posts)
PATCH	    /users/me/profile Logged in	Update own profile (1:1)
DELETE	    /users/:id	    Admin	Delete a user
GET	        /categories  	Public	List categories
POST	    /categories 	Admin	Create category
DELETE	    /categories/:id	Admin	Delete category
GET	        /tags	       Public	List tags
POST	    /tags	       Admin	Create tag
DELETE	    /tags/:id	   Admin	Delete tag
GET	        /posts	       Public	List published posts
GET	        /posts/:slug   Public	Post detail (author, tags, comments)
POST	    /posts	       Logged in	Create post (+ attach tags N:N)
PATCH	    /posts/:id	   Owner/Admin	Update post
DELETE	    /posts/:id	   Owner/Admin	Delete post
POST	    /posts/:postId/comments	Logged in	Add comment
DELETE	    /posts/:postId/comments/:commentId	Owner/Admin	Delete comment
Send the JWT either as `Authorization: Bearer <token>` or let the browser
send it automatically via the httpOnly `auth` cookie set on login/register.
